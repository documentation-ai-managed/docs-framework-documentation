## Campaign Scheduling

GetFramework's intelligent scheduling system helps you optimize content delivery across multiple channels and time zones. Our AI analyzes engagement patterns to suggest optimal posting times for maximum reach.

<Callout kind="tip">Scheduling campaigns in advance ensures consistent brand presence and frees up your team for strategic work.</Callout>

<Steps>
  <Step title="Create a Campaign" icon="plus">
    Start by selecting a template and defining your target audience and channels.
  </Step>
  <Step title="Set Schedule" icon="calendar">
    Choose between one-time posts, recurring schedules, or AI-optimized timing.
    
    <CodeGroup tabs="JavaScript,Python">
      ```javascript
// Schedule a one-time campaign
await client.campaigns.schedule({
  content: 'New product announcement',
  channels: ['facebook', 'twitter'],
  schedule: {
    type: 'one-time',
    datetime: '2024-10-20T14:00:00Z'
  }
});
```
      ```python
# Create a recurring schedule
schedule = client.schedules.create({
    name: 'Weekly Newsletter',
    frequency: 'weekly',
    day_of_week: 'monday',
    time: '09:00',
    timezone: 'America/New_York'
})
```
    </CodeGroup>
  </Step>
  <Step title="Review and Publish" icon="check-circle">
    Preview your scheduled content and publish it to go live at the specified time.
  </Step>
</Steps>

## Multi-Location Scheduling

For franchises with multiple locations, GetFramework supports coordinated scheduling that respects local time zones and market conditions.

<Columns cols={2}>
  <Card title="Central Control" icon="settings" href="#">
    Set global schedules from headquarters while allowing local customization.
  </Card>
  <Card title="Local Adaptation" icon="map-pin" href="#">
    Each location can adjust timing based on their specific market needs.
  </Card>
</Columns>

<Expandable title="Location-Based Scheduling" default-open="false">
  ```javascript
// Schedule content for multiple locations
await client.campaigns.schedule({
  content: 'Local promotion',
  locations: [
    { id: 'store-nyc', timezone: 'America/New_York', offset: 0 },
    { id: 'store-la', timezone: 'America/Los_Angeles', offset: -3 }
  ],
  schedule: {
    type: 'optimized',
    baseTime: '14:00'
  }
});
```
</Expandable>

## Troubleshooting Common Issues

- **Posts not publishing**: Check time zone settings and ensure the account has necessary permissions
- **Inconsistent timing**: Verify that your system's clock is synchronized
- **Low engagement**: Consider adjusting posting times or content types based on analytics

<Expandable title="Schedule Conflicts" default-open="false">
  GetFramework automatically detects and resolves scheduling conflicts. If two campaigns are scheduled for the same time, the system prioritizes based on campaign importance and adjusts timing as needed.
</Expandable>
